Manual de Usuario: Estructura de Carpetas
1. client/
Contiene el código del frontend (React). Aquí se desarrollan las interfaces de usuario y la lógica de interacción con el backend.

public/: Archivos públicos y estáticos (imágenes, íconos, etc.).
src/: Código fuente principal del frontend.
api/: Funciones para interactuar con el backend (peticiones HTTP).
assets/: Recursos gráficos (imágenes, íconos).
components/: Componentes reutilizables de la interfaz.
context/: Contextos de React para manejar estados globales.
pages/: Páginas principales de la aplicación (vistas).
schemas/: Validaciones de formularios.
Archivos de configuración: vite.config.js, tailwind.config.cjs, etc.
2. src/
Contiene el código del backend (Node.js + Express). Aquí se gestiona la lógica de negocio, la base de datos y las rutas de la API.

controllers/: Lógica para manejar las peticiones de la API.
libs/: Librerías auxiliares (por ejemplo, manejo de JWT).
middlewares/: Funciones intermedias para validar y proteger rutas.
models/: Definición de los modelos de datos (MongoDB).
routes/: Definición de las rutas de la API.
schemas/: Validaciones de datos para la API.
tests/: Pruebas automáticas del backend.
3. requests/
Archivos para probar las rutas de la API usando herramientas como REST Client.

4. Archivos raíz
docker-compose.yml: Configuración para ejecutar el proyecto con Docker.
package.json: Dependencias y scripts del proyecto.
README.md: Documentación básica del proyecto.
